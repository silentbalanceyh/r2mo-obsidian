添加项目（Vault）时，在文件管理器中对可打开的文件进行一个判断（必须是 .obsidian 目录存在的地方可以被打开），其他地方不可以被打开，因为只有 .obsidian 是一个 obsidian 的 Vault 仓库，还有，隐藏目录类似 .r2mo 这种也应该可以呈现，而不需要 MacOS 或 Windows 直接显示隐藏目录才可以。

## Changes
- 2026-03-30: [Orchestrator] Completed task - 强化文件管理器
  - Vault 验证: Add 选择目录时检查 .obsidian 目录，无效目录弹出警告并拒绝添加
  - 显示隐藏目录: 文件选择对话框使用 DontUseNativeDialog + QDir::Hidden 过滤器
    - 用户可在选择时看到 .obsidian 目录，确认是有效 Vault
    - 同时显示 .r2mo 等其他隐藏目录
  - 窗口尺寸: 默认分辨率调整为 1680 x 900
  - 两栏布局: Vaults 列表 | 预览面板
  - 预览信息: 显示文件数、文件夹数、隐藏项目数，检测 .obsidian 和 .r2mo 目录
