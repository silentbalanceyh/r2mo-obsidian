# 状态计算精修 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将实时监控板中 Claude、Codex、OpenCode 的 `Working/Ready` 判定收敛到工具自身事件语义，减少仅凭 terminal 进程、CPU 抖动与 artifact mtime 导致的误判。

**Architecture:** 保留 terminal/root-process 检测用于“是否在线”和窗口定位，但把状态判定提升为“工具事件优先、弱启发兜底”的两层状态机。`SessionScanner` 负责本地事件语义解析，`RemoteSessionScanner` 负责远程 probe 对齐同一套语义，监控板只消费统一的 `SessionStatus` 结果。

**Tech Stack:** Qt 6.10.0, C++17, shell guard tests, local artifact parsing (`~/.claude/projects`, `~/.codex/sessions`, `.omc/sessions`, `opencode.global.dat`)

- [ ] **Step 1: 写本地状态机目标**

```cpp
// Claude:
//   tool_use / PreToolUse / PostToolUse => Working
//   end_turn / Stop / hook_success => Ready
//
// Codex:
//   task_started / exec_command_begin / function_call_begin / patch_apply_begin / agent_reasoning => Working
//   task_complete / user_message / final_answer / function_call_output / patch_apply_end => Ready
//
// OpenCode:
//   global notification turn-complete / error / paused => Ready
//   session artifact ended_at / endedAt / reason => Ready
//   no explicit active event => Unknown, let live process + keepalive decide
```

- [ ] **Step 2: 最小实现 OpenCode 全局通知优先解析**

```cpp
if (type == "turn-complete" || type == "error") {
    return SessionStatus::Ready;
}
if (type == "turn-start" || type == "session.status") {
    return SessionStatus::Working;
}
```

- [ ] **Step 3: 收紧 CPU/mtime 兜底，只在 artifact 语义未知且进程真实在线时使用**

```cpp
if (artifactStatus != SessionStatus::Unknown) {

}
if (!isRunning) {
    return SessionStatus::Ready;
}
```
